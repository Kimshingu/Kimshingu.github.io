장바구니 페이지(order/cart)-----------------cart 테이블
- 해당 user가 로그아웃해도 장바구니 정보는 남아있는다.

*상품 페이지*에서 [장바구니에 담기] 버튼을 누르면
1. 이미지??, 상품명, 사이즈, 색상, 수량, 가격 정보가 
   하나의 insert문으로 cart 테이블에 row로 추가된다.
2. cart테이블의 row를 *장바구니 페이지*에 구현한다.
3. 각각의 row의 가격을 합하여 총 가격을 구현한다.
4. [계속 쇼핑하기] 버튼을 누르면 홈화면으로 간다.
 4-1. 홈화면의 카트 아이콘에 수량이 추가된다.(유지된다.)
5. [결제하기] 버튼을 누르면 구매하기 페이지로 간다.

구매하기 페이지(order/cart_order_form)------order 테이블
*상품 페이지*에서 [바로구매] 버튼을 누를 경우

   product 테이블의 이미지??, 상품명, 사이즈, 색상, 수량, 가격
   user 테이블의 주문자, 휴대폰, 이메일
   받는사람 정보는 직접 입력
   가격 + 배송비 총 가격표시
   결제 방법 선택
   [결제하기] 버튼을 누르면 order 테이블에 저장되며
   팝업 창으로 해당 정보 띄우기

*장바구니 페이지*에서 [구매하기] 버튼을 누를경우 
   cart 테이블의 row들,
   user 테이블의 주문자, 휴대폰, 이메일
   받는사람 정보는 직접 입력
   가격 + 배송비 총 가격표시
   결제 방법 선택
   [결제하기] 버튼을 누르면 order 테이블에 저장되며
   팝업 창으로 해당 정보 띄우기
